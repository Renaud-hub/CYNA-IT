# 🔍 Tests techniques — CYNA‑IT

Ce dossier contient l’ensemble des **tests techniques** réalisés dans le cadre du projet CYNA‑IT.  
Il regroupe les analyses réseau, les scans de vulnérabilités et les preuves techniques nécessaires au rapport final.

---

## 📄 Contenu du dossier

### **1. Tests Nmap**
Inclut :
- scan des ports ouverts  
- détection des services  
- identification des versions  
- analyse de la surface d’attaque  
- preuves techniques (captures, logs)

Ces tests permettent d’évaluer l’exposition réseau des machines du projet.

---

### **2. Tests OWASP ZAP**
Inclut :
- scan de vulnérabilités web  
- détection des failles (XSS, injections, headers manquants, etc.)  
- analyse du comportement HTTP  
- preuves techniques (rapports HTML/PDF)

Ces tests permettent d’évaluer la sécurité des applications web exposées.

---

### **3. Autres tests techniques**
Selon le projet, ce dossier peut aussi contenir :
- résultats de scans automatisés  
- scripts de test  
- captures d’écran  
- logs d’outils de sécurité  

---

## 🎯 Objectif du dossier

- Centraliser les preuves techniques du projet  
- Documenter les tests réseau et applicatifs  
- Faciliter la relecture et la validation du rapport final  
- Fournir un historique clair des analyses réalisées
