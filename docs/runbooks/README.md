# 📚 Runbooks — CYNA‑IT

Ce dossier contient l’ensemble des **runbooks opérationnels** du projet CYNA‑IT.  
Un runbook est une procédure claire, étape par étape, permettant de gérer un incident, une alerte ou une action d’exploitation.  
Ils sont essentiels pour garantir une réponse rapide, standardisée et sans erreur.

---

## 📄 Contenu du dossier

### **1. Runbook — Incident Response**
Décrit les actions à mener lorsqu’un incident de sécurité est détecté :
- identification de l’incident  
- confinement  
- analyse technique  
- résolution  
- documentation finale  

Ce runbook sert de base pour toute intervention SOC.

---

### **2. Runbook — Gestion des alertes Wazuh**
Procédure pour traiter les alertes générées par Wazuh :
- analyse de l’alerte  
- vérification des logs  
- classification (false positive / true positive)  
- actions correctives  
- mise à jour des règles si nécessaire  

---

### **3. Runbook — Investigation Suricata**
Procédure d’analyse des événements IDS :
- lecture des événements EVE JSON  
- corrélation avec les logs système  
- identification des signatures déclenchées  
- actions de mitigation réseau  

---

### **4. Runbook — Backup & Restore**
Procédure pour gérer les sauvegardes et restaurations :
- sauvegarde des données critiques  
- restauration en cas d’incident  
- vérification de l’intégrité  
- documentation des opérations  

---

## 🎯 Objectif du dossier

- Standardiser les actions d’exploitation  
- Réduire les erreurs humaines  
- Accélérer la réponse aux incidents  
- Documenter les procédures critiques  
- Faciliter l’onboarding des nouveaux membres
